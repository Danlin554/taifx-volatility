## 本次踩到的坑

1. **`_make_xnys` mock 要用 `pd.DatetimeIndex`**：Python list 沒有 `.tolist()` 方法，被 `last_completed_us_session` 的 `sessions.tolist()` 呼叫時拋 AttributeError，被 except 吃掉回 None，測試靜默失敗。
2. **mock `pd.read_sql` 返回不應預先 set_index**：真實 `pd.read_sql` 返回 `trade_date` 是欄位（column），`db.read_indices_until` 才做 `set_index`；mock 要模擬原始輸出格式。
3. **`_patch_both_calendars` sessions 同樣問題**：同上，`sessions_in_range.return_value` 須是 `pd.DatetimeIndex`。
4. **2026 年春節日期**：計畫文件寫錯（誤植為 1月），實際春節 2/17，TWSE 休市 2/12~2/20，整合測試要用真實日期驗證。
5. **`upsert_indices_batch` 測試需 `patch.dict(os.environ, {"DATABASE_URL": ...})`**：函式直接讀 `os.environ["DATABASE_URL"]` 作為 `psycopg2.connect` 的參數，mock `psycopg2.connect` 前環境變數已被求值。

## 關鍵架構決策（本次確認）

- **Live publish 嚴格規則**：NYSE D 未收盤絕不 publish `effective_date=D`，fail-closed 設計
- **Override flags 拆分**：TXF freshness 與 US session 各自獨立 opt-in，避免過度降級
- **四階段 publish 原子策略**：render_tmp（.tmp 檔）→ db_upsert → rename_html（POSIX atomic）→ update_cache；接受 db_upsert 成功後 process crash 的 <1 秒窗口（startup refresh 自動收斂）
- **`expected_us_session_for_effective_date` 修正**：用 `XTAI.next_session(D)` 而非 `D+1` 曆天，正確處理台灣長假情境
